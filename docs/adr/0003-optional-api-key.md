# ADR 0003：采用可选 API Key 保护写操作和 Agent 接口

## 状态

已采纳

## 背景

CodePilot 的部分接口具有副作用或成本：

- 导入仓库、索引、重新索引和删除记录会改变系统状态。
- Chat、分析、Review 和生成类接口可能触发模型调用。
- 代码检索和 Agent 接口可能暴露仓库上下文。

同时，项目需要保持本地演示足够简单。如果强制所有环境都配置鉴权，会增加面试演示和本地调试成本。

## 决策

采用可选 API Key：

- `CODEPILOT_API_KEY` 为空时，不启用鉴权，方便本地演示。
- `CODEPILOT_API_KEY` 配置后，写操作、代码检索和 Agent 接口需要请求头 `X-CodePilot-Api-Key`。
- 健康检查、系统诊断、仓库列表、源码浏览、符号查看和运行历史查询保持公开。
- 诊断接口只暴露模型是否配置，不返回任何密钥内容。

## 后果

优点：

- 本地演示门槛低。
- 部署环境具备最小安全边界。
- 安全策略清晰，测试覆盖开启、关闭和错误 Key 场景。

代价：

- 这不是完整用户认证体系。
- 无法区分不同用户和权限角色。
- 公共只读接口在生产环境可能还需要更细粒度保护。

## 后续演进

如果项目进入多人使用场景，可以引入用户体系、角色权限、仓库级访问控制和审计日志。当前阶段不建议为了面试演示过早引入复杂认证。
