# ADR 0001：采用离线优先的 Agent 运行策略

## 状态

已采纳

## 背景

CodePilot 需要在本地开发、CI 和面试演示环境中稳定运行。真实 LLM 调用会受到 API Key、网络、模型服务可用性和调用成本影响。如果系统完全依赖外部模型，测试和演示链路会变得不稳定。

项目当前的核心目标是展示代码仓库理解闭环，而不是证明某一个模型服务始终可用。

## 决策

CodePilot 采用离线优先策略：

- 检索、解析、索引和运行历史都可以在本地完成。
- 配置 DeepSeek API Key 后，Agent 调用真实模型生成回答。
- 未配置 API Key 或模型调用失败时，系统返回基于检索上下文的本地回退回答。
- 测试默认不依赖真实模型调用。

## 后果

优点：

- CI 稳定，不依赖外部网络和模型服务。
- 面试现场即使没有模型 Key，也能展示导入、索引、检索、citations 和运行历史。
- 可以明确区分“工程闭环能力”和“模型生成质量”。

代价：

- 本地回退回答不代表真实模型效果。
- 需要在文档和演示中说明当前回答可能来自本地回退。

## 后续演进

后续可以增加更细的模型调用状态，例如在响应中标记 `llm_mode=remote/fallback`，或者在 RunStore 中记录模型调用失败原因。
